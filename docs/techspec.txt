# AgentNode.net – Technical Implementation Spec v3 Final
# Claude Code Ready · Phase A + B (MVP)
# März 2026
## 1. PROJEKT-ÜBERSICHT
AgentNode ist eine Registry + Resolution Engine für installierbare AI-Agent-Capabilities.
Entwickler veröffentlichen ANP-Pakete (AgentNode Packages), andere Entwickler suchen,
prüfen und integrieren sie über CLI, SDK oder Web.
Lokale Paketinstallation erfolgt im MVP ausschließlich über die CLI.
Das Python SDK dient im MVP für Search, Resolution, Policy-Checks und Install-Metadaten,
nicht für lokales pip install.
Kernflow: Capability Gap → Resolution → Install
## 2. TECH-STACK (final, keine Alternativen)
```
Backend:      Python 3.12 + FastAPI + Uvicorn
ORM:          SQLAlchemy 2.0 (async only, AsyncSession)
Migrations:   Alembic
Datenbank:    PostgreSQL 16 (async via asyncpg)
Cache:        Redis 7
Suche:        Meilisearch v1.6
Storage:      S3-kompatibel (Cloudflare R2 prod, MinIO lokal)
Auth:         JWT (access + refresh, stateless) + API Keys (SHA-256 hash) + TOTP (2FA)
Validation:   Pydantic v2
CLI:          Node.js 20+ (TypeScript) → npm package "agentnode"
SDK:          Python → PyPI package "agentnode-sdk"
Frontend:     Next.js 14 (React, TypeScript, Tailwind CSS)
Testing:      pytest + httpx (Backend), vitest (CLI)
CI/CD:        GitHub Actions
Hosting:      Fly.io (Backend) + Vercel (Frontend) + Cloudflare R2 (Artifacts)
```
WICHTIG – Zwei Installationspfade:
```
CLI für Entwickler:     npm install -g agentnode
Python SDK für Code:    pip install agentnode-sdk
```
Diese MÜSSEN überall konsistent unterschieden werden.

## 4.1 VERSION LIFECYCLE RULES
```
latest_version_id semantics:
- MUST point to the newest PUBLIC, STABLE, non-yanked, non-quarantined version
- If NO such version exists → latest_version_id MUST be NULL
- MUST be recalculated in the SAME transaction when ANY of these happen:
  - publish
  - deprecate
  - yank / unyank
  - clear quarantine
  - reject version
Definitions:
- public version  = quarantine_status IN ('none', 'cleared') AND is_yanked = false
- stable version  = channel = 'stable'
- latest_version_id = newest public + stable version ordered by published_at DESC
                      Returns NULL if no qualifying version exists.
Deprecation:
- is_deprecated is on the PACKAGE level, not per-version
- deprecated means: the whole package is considered outdated
- individual versions are removed via yanking, not per-version deprecation
- no per-version deprecation state in MVP
```
## 4.2 VERSION SELECTION HELPERS
```
Implement shared query helpers in backend/app/packages/version_queries.py:
get_public_versions(package_id) → list
  Returns: stable/beta versions WHERE quarantine_status IN ('none','cleared')
           AND is_yanked = false, ordered by published_at DESC

### 8.1.1 JWT TOKEN SPEC
```
Token model (MVP):
- access_token and refresh_token are BOTH JWTs
- access_token claims: { sub: user_id, token_type: "access", exp: <short> }
- refresh_token claims: { sub: user_id, token_type: "refresh", exp: <long> }
- Same JWT_SECRET for both (acceptable in MVP)
- Access token expiry: JWT_ACCESS_TOKEN_EXPIRE_MINUTES (default 60)
- Refresh token expiry: JWT_REFRESH_TOKEN_EXPIRE_DAYS (default 30)
- No refresh token rotation in MVP
- No server-side token revocation in MVP
- /v1/auth/refresh MUST validate token_type == "refresh" before issuing new access token
```
### 8.1.2 API KEY VALIDATION FLOW
```
API key format: ank_<random_secret> (full key ~40 chars)
key_prefix: first 8 chars stored for fast DB lookup, e.g. "ank_ab12"
Validation flow:
1. Extract key from X-API-Key header
2. Extract prefix (first 8 chars)
3. Query api_keys table by key_prefix (may return multiple candidates)
4. Compute SHA-256 of full provided key
5. Compare against key_hash_sha256 of each candidate
6. If match found AND revoked_at IS NULL → authenticated
7. If revoked_at IS NOT NULL → reject with AUTH_API_KEY_REVOKED (401)
8. If no match → reject with AUTH_API_KEY_INVALID (401)
9. On success: update last_used_at (async or inline, acceptable to skip in hot path)
Notes:
- API keys are user-scoped (inherit all permissions of owning user)
- API keys do NOT bypass ownership checks
- API keys do NOT bypass 2FA requirement for publish
```

## 9. RESOLUTION ENGINE
```python
# backend/app/resolution/engine.py
WEIGHTS = {
    "capability_match": 0.40,
    "framework_match": 0.20,
    "runtime_match": 0.15,
    "trust_score": 0.15,
    "permissions_fit": 0.10,
}
TRUST_SCORES = {
    "curated": 1.0, "trusted": 0.8, "verified": 0.6, "unverified": 0.3,
}
def score_package(pkg_version, request) -> float | None:
    """Returns score 0-1 or None if excluded."""
    # --- HARD EXCLUSIONS ---
    if pkg_version.is_yanked:
        return None
    if pkg_version.quarantine_status not in ("none", "cleared"):
        return None
    # Policy hard blocks
    if request.policy:
        pkg_trust = pkg_version.package.publisher.trust_level
        if request.policy.min_trust and TRUST_SCORES.get(pkg_trust, 0) < TRUST_SCORES.get(request.policy.min_trust, 0):
            return None
        if not request.policy.allow_shell and pkg_version.permissions.code_execution_level == "shell":
            return None
        if not request.policy.allow_network and pkg_version.permissions.network_level == "unrestricted":
            return None
    # --- HARD MATCHING ---
    # Capability: HARD requirement
    cap_ids = [c.capability_id for c in pkg_version.capabilities]
    if request.missing_capability not in cap_ids:
        return None
    # Runtime: HARD requirement if specified
    if request.runtime and pkg_version.runtime != request.runtime:
        return None
    # Framework: HARD requirement if specified (unless "generic" is listed)
    frameworks = [r.framework for r in pkg_version.compatibility_rules]
    if request.framework and request.framework not in frameworks and "generic" not in frameworks:
        return None
    # --- SOFT SCORING ---
    s_cap = 1.0  # already validated above
    s_fw = 1.0   # already validated above
    s_rt = 1.0   # already validated above
    s_trust = TRUST_SCORES.get(pkg_version.package.publisher.trust_level, 0.3)
    perm = pkg_version.permissions
    s_perm = 1.0
    if perm.network_level == "unrestricted": s_perm -= 0.3
    if perm.code_execution_level == "shell": s_perm -= 0.4
    if perm.filesystem_level in ("workspace_write", "any"): s_perm -= 0.2
    s_perm = max(0.0, s_perm)
    # Deprecated packages get a penalty but are NOT excluded
    deprecated_penalty = 0.15 if pkg_version.package.is_deprecated else 0.0
    total = (s_cap * WEIGHTS["capability_match"] + s_fw * WEIGHTS["framework_match"] +
             s_rt * WEIGHTS["runtime_match"] + s_trust * WEIGHTS["trust_score"] +
             s_perm * WEIGHTS["permissions_fit"]) - deprecated_penalty
    return round(max(0.0, total), 4)
```

## 11. TYPOSQUATTING DETECTION
```python
from difflib import SequenceMatcher
def check_typosquatting(new_slug: str, existing_slugs: list[str]) -> list[str]:
    suspicious = []
    norm_new = new_slug.replace("-", "").replace("_", "")
    for existing in existing_slugs:
        if new_slug == existing:
            continue
        if SequenceMatcher(None, new_slug, existing).ratio() > 0.85:
            suspicious.append(existing)
        if norm_new == existing.replace("-", "").replace("_", ""):
            suspicious.append(existing)
    return list(set(suspicious))
# If suspicious: set quarantine_status = "quarantined" with reason "typosquatting_suspected"

### 13.2.1 Lockfile Conflict Behavior
```
When installing a package that already exists in agentnode.lock:
Case 1 — same slug, same version:
  → skip installation, print "pdf-reader-pack@1.0.0 already installed"
Case 2 — same slug, different version (install or update):
  → perform installation of new version
  → update lockfile entry with new version, hash, timestamp
  → print "pdf-reader-pack upgraded 1.0.0 → 1.1.0"
Case 3 — rollback to specific version:
  → perform installation of target version
  → replace lockfile entry with target version
  → print "pdf-reader-pack rolled back to 1.0.0"
```
### 13.3 Python Interpreter Selection
```
CLI Python resolution order:
1. If VIRTUAL_ENV env var is set → use $VIRTUAL_ENV/bin/python (or Scripts\python.exe on Windows)
2. Else if .venv/bin/python exists in current directory → use it
3. Else try python3
4. Else try python
5. If none found → fail with clear error:
   "No Python interpreter found. Please activate a virtual environment or ensure python3 is on PATH."
The CLI MUST print which Python interpreter is used when --verbose is set.
The CLI MUST use the resolved Python for: python -m pip install <path>
Important:
- CLI always installs into the currently active/detected Python environment
- CLI NEVER creates virtual environments automatically
- If no venv is active, packages are installed into the system/user Python (user's responsibility)
```
### 13.4 Install Flow (technical)
```
1. POST /v1/packages/{slug}/install (with event_type "install"|"update"|"rollback") → get presigned artifact_url + hash
2. Download artifact to temp directory
3. Verify SHA256 hash matches API-returned hash (FAIL if mismatch, delete temp, abort)
4. Validate archive:
   - reject path traversal (../)
   - reject symlinks
   - reject nested archives
   - reject individual files > 10MB
   - reject archives with > 500 files
   - reject if no agentnode.yaml at archive root
   - reject if no pyproject.toml at archive root
5. Extract tar.gz to temp dir
6. Verify entrypoint module path exists in extracted files
7. Resolve Python interpreter (section 13.3)
8. Run: <resolved_python> -m pip install <temp_dir_path>
9. On success: write to agentnode.lock (incl package_type + entrypoint), print post_install_code
10. On failure: print error with clear message
11. ALWAYS clean up temp dir regardless of success/failure
```

HTTP timeout: 30 seconds for all API calls
Download timeout: 120 seconds for artifact download
Retries: NONE in MVP. No automatic retry on any error.
On timeout: print clear error "Request timed out. Please try again."
On 5xx: print "Server error (status). Please try again later."
On network failure: print "Network error: <details>. Check your connection."
Future: configurable retry for downloads (transient network errors only)

```python
# sdk/agentnode_sdk/client.py
import httpx
from .exceptions import AgentNodeError, NotFoundError, AuthError, ValidationError
ERROR_CLASS_MAP = {
    401: AuthError,
    403: AuthError,
    404: NotFoundError,
    422: ValidationError,
}
class AgentNode:
    def __init__(self, api_key: str, base_url: str = "https://api.agentnode.net/v1"):
        self._client = httpx.Client(base_url=base_url, headers={"X-API-Key": api_key}, timeout=30)
    def search(self, query: str = "", capability_id: str = "", framework: str = "",
               sort_by: str = "relevance", page: int = 1) -> dict:
        return self._post("/search", query=query, capability_id=capability_id,
                          framework=framework, sort_by=sort_by, page=page)
    def resolve_upgrade(self, missing_capability: str, framework: str = "", runtime: str = "",
                        current_capabilities: list[str] = [], policy: dict = {}) -> dict:
        return self._post("/resolve-upgrade", missing_capability=missing_capability,
                          current_capabilities=current_capabilities, framework=framework,
                          runtime=runtime, policy=policy)
    def check_policy(self, package_slug: str, framework: str = "", policy: dict = {}) -> dict:
        return self._post("/check-policy", package_slug=package_slug, framework=framework, policy=policy)
    def get_install_metadata(self, package_slug: str, version: str = "") -> dict:
        """Request install metadata from AgentNode WITHOUT side effects.
        Calls GET /install-info (read-only). Does NOT create installation records
        or increment download counts. Does NOT perform local pip installation.
        Local installation is handled by the CLI (agentnode install)."""
        params = {"version": version} if version else {}
        return self._handle(self._client.get(f"/packages/{package_slug}/install-info",
                                             params=params))
    def get_package(self, slug: str) -> dict:
        return self._handle(self._client.get(f"/packages/{slug}"))
    def validate(self, manifest: dict) -> dict:
        return self._handle(self._client.post("/packages/validate", json={"manifest": manifest}))
    def _post(self, path: str, **kwargs) -> dict:
        return self._handle(self._client.post(path, json={k: v for k, v in kwargs.items() if v}))
    def _handle(self, response) -> dict:
        """Parse response. Raise typed AgentNodeError on API errors."""
        if response.status_code >= 400:
            try:
                body = response.json()
                err = body.get("error", {})
                code = err.get("code", "UNKNOWN")
                message = err.get("message", response.text)
            except (ValueError, KeyError):
                code, message = "UNKNOWN", response.text
            exc_class = ERROR_CLASS_MAP.get(response.status_code, AgentNodeError)
            raise exc_class(code, message)
        return response.json()
```

from agentnode_sdk import AgentNode
from langchain_core.tools import StructuredTool
import importlib
def load_tool(package_slug: str, api_key: str = "", version: str = "") -> StructuredTool:
    """Load an installed ANP package as a LangChain StructuredTool.
    IMPORTANT: The package MUST already be installed via `agentnode install <slug>`.
    This adapter does NOT auto-install packages. If the package is not found locally,
    it raises ImportError with instructions to install via CLI.

    client = AgentNode(api_key=api_key)
    pkg = client.get_package(package_slug)
    entrypoint = pkg["blocks"]["install"]["entrypoint"]
    try:

    except ImportError:
        raise ImportError(
            f"Package '{package_slug}' is not installed locally. "
            f"Run: agentnode install {package_slug}"
        )
    cap = pkg["blocks"]["capabilities"][0]
    return StructuredTool.from_function(
        func=module.run,
        name=cap["name"],
        description=cap["description"],
    )

### 20.1 Frontend API Proxy Pattern
```
Single catch-all route handler: /api/[...path]/route.ts
Flow:
1. Browser calls /api/packages/pdf-reader-pack
2. Next.js route handler reads httpOnly auth cookie
3. Forwards request to backend: https://api.agentnode.net/v1/packages/pdf-reader-pack
   with Authorization: Bearer <token> header
4. Returns backend response to browser
Implementation:
- ONE catch-all route handler, NOT separate handlers per endpoint
- Reads access_token from httpOnly cookie, sets Authorization header
- If no cookie present, forwards request without auth (for public endpoints)
- Passes through query params, request body, and Content-Type
- Returns backend status code and response body as-is
```

## 23. BACKGROUND TASKS & FAILURE HANDLING
### 23.1 Security Scanner
MVP uses **FastAPI BackgroundTasks** (in-process async) for security scanning after publish.
This is NOT durable across process restarts. Acceptable for MVP.
Future: move to Redis queue / Celery worker.
```python
# In publish endpoint:
from fastapi import BackgroundTasks
@router.post("/packages/publish")
async def publish(bg: BackgroundTasks, ...):
    # ... create package, upload artifact ...
    bg.add_task(run_security_scan, version_id=version.id)
    return {"package_id": ..., "version_id": ...}
```
### 23.2 Meilisearch Sync Failure
```
DB is the source of truth. ALWAYS.
Publish flow:
1. DB transaction commits FIRST
2. Meilisearch sync happens AFTER commit
3. If Meilisearch sync fails:
   - Log error (logger.error with package slug and version)
   - Package remains valid and accessible via direct API (DB-backed)
   - DO NOT set a DB flag in MVP. Use logging only.
   - Search may be temporarily stale until manual reindex
   - Install/detail/resolution endpoints remain authoritative via DB
4. scripts/reindex_search.py can be run manually to fix stale index
   (queries all non-quarantined, non-yanked packages from DB and re-syncs to Meilisearch)
```
### 23.3 Artifact Validation (expanded)
After archive extraction, BEFORE pip install:
```
1. agentnode.yaml MUST exist at archive root
2. pyproject.toml MUST exist at archive root
3. pyproject.toml project.version MUST match manifest version
4. Entrypoint module path MUST exist in extracted files
   (e.g. if entrypoint is "pdf_reader_pack.tool", then pdf_reader_pack/tool.py must exist)
5. No path traversal (../)
6. No symlinks
7. No nested archives (.tar.gz, .zip inside)
8. No individual file > 10MB
9. No more than 500 files total
10. Total archive < 50MB
```
### 23.4 WEB AUTH MECHANISM
```
Web frontend stores auth via secure httpOnly cookies through Next.js server-side route handlers.
Flow:
1. User submits login form → Next.js /api/auth/login route handler
2. Route handler calls backend POST /v1/auth/login
3. Backend returns JWT access_token + refresh_token
4. Route handler sets both as httpOnly, secure, sameSite=strict cookies
5. Subsequent frontend API calls go through /api proxy which reads cookie and forwards as Bearer header
6. Browser JavaScript NEVER has direct access to JWTs
7. Browser JavaScript MUST NOT store JWTs in localStorage or sessionStorage
Logout:
- Next.js /api/auth/logout clears cookies
- No server-side token revocation in MVP
- Compromised refresh tokens remain valid until expiry (documented limitation)
Session expiry:
- Refresh tokens expire after JWT_REFRESH_TOKEN_EXPIRE_DAYS (default 30 days)
- After expiry: user is automatically logged out and must re-authenticate
- No "remember me" or session extension in MVP
```
